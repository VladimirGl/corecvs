00292 /**
00293  *  As the transformation procedure uses the inverse of the transformation matrix
00294  *  We first apply the rectification matrix, and then the scaling and shift.
00295  *  With inverses it is vice versa
00296  *
00297  *  We have a description of the math here - corecvs::StereoAligner::getAlignmentTransformation()
00298  *
00299  *
00300  *
00301  *  Let's have a look starting form camera input to rectification input image
00302  *
00303  *  1. CS = First we shift image by half to put it's center to [0,0]
00304  *  2. MI = Second we make mirror and rotation operation
00305  *  3. BH = Then we shift it back to where it was
00306  *  4. SH = We make a shift of the mirrored image
00307  *  5. SL = Afterwards we scale
00308  *
00309  *  Total transformation
00310  *     T1 = SL * SH * (BH * MI * CS)
00311  *
00312  *  as BH = CS^(-1)
00313  *     T1^(-1) = BH * MI * CS * SH * SL
00314  *
00315  *  T1       transforms from camera image to rectification input image
00316  *  T1^(-1)  transforms from rectification input image to camera image
00317  *
00318  *  following the operation from corecvs::StereoAligner we have a rectification transformation for
00319  *  camera images - H_left and H_right.
00320  *
00321  *  H_left       - from camera input to rectified image
00322  *  H_left^(-1)
00323  *
00324  *
00325  *
00326  *
00327  *
00328  **/
