template<typename IndexType>
static IndexType corecvs::BlockedRange< IndexType >::do_split ( BlockedRange< IndexType > &  r) [inline, static]

Please not the design here -.

During the split the input range r is transformed into the first part of the split, and the newly constructed object becomes the second part.

Attention:
For a newly created part to finish at the appropriate place we need to _first_ initialize the myEnd , and then myBegin.

TBB blocked_range<Type> version does it by defining end before begin. I don't think one should give up semantics for precisely following C++ guidelines of using initialization list.
